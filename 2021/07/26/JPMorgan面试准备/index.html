      Q: Have you worked with Java Collections and if so, what is it used for?

  • 概括地说，Collection就是一个通用的数据结构接口，这个接口提供按照index查询数据的能力，其中数据有序，且可以重复.

  • 但是JDK不允许直接实现Collection接口，因此也定义了三个子接口list，set和queue，实现类会实现这些子接口的方法

  • List是有序，且可以重复的数据接口，实现类有linkedlist 和 arraylist

    • LinkedList 底层是双向链表，查询慢 O(n)，增删块 O(1), linkedlist 不仅可以作为list来用，它同时满足了deque和queue的定义
    • ArrayList 底层是数组，查询快O(1)，增删慢 O(n)

  • Set 是无序，且不允许重复的数据结构

    • HashSet: 底层是一个HashMap，可以有添加，删除，以及查看元素是否在set内的操作
    • TreeSet: 底层是TreeMap，排序时候会用

  • queue 是先进先出的数据结构 一般情况下linkedlist可以实现queue的全部功能，但是还有一些其他的值得说的队列

    • 优先级队列： PriorityQueue通过二叉小顶堆（min heap)实现，优先级队列保证每次取出的队列都是权值最小的

    • 阻塞队列 ：阻塞队列是一个可以阻塞的先进先出集合，比如某个线程在空队列获取元素时、或者在已存满队列存储元素时，都会被阻塞。

      • ArrayBlockingQueue ：基于数组的有界阻塞队列，必须指定大小。
      • LinkedBlockingQueue ：基于单链表的无界阻塞队列，不需指定大小。
      • PriorityBlockingQueue ：基于最小二叉堆的无界、优先级阻塞队列。

  • 还有一个数据结构并不实现Collection接口，但是也非常常见，就是Map，通过键值对来保存数据。Map的详细区分参考Java面试相关。

  • 另外值得一提的是，上面的接口都是线程不安全的，在多线程场景中需要使用线程安全的数据结构,如ConcurrentHashMap，SynchronizedList等

What do you mean by data encapsulation?

就是把数据放到类里面去，然后用getter和setter把数据拿出来。

优点:

  • 保护数据
  • 提升内聚，降低耦合
  • 可以对成员变量进行更精确的控制，如在getter setter中添加验证
  • 隐藏细节
